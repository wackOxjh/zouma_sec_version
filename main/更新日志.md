# 更新日志

所有显著的变更都会记录在这个文件夹中。

## \[2.1.0] - 2026-04-27

**新增：**

- 新增`lq_reg_uart`程序，采用寄存器的方式来初始化串口和串口数据收发
- 新增`lq_reg_uart`相关的示例程序

**移除：**

- 移除了`lq_uart`相关程序，该程序使用设备文件进行操作，拓展其他串口需要修改内核，较为麻烦

## \[2.1.0] - 2026-04-22

- 在`lq_reg_pwm`的类中继承了`lq_auto_cleanup`基类，使其创建变量后自动注册到清理容器中，可使程序无论是正常退出还是异常退出都可以执行清理函数

## \[2.1.0] - 2026-03-05

**更改：**

- 更新`build.sh`文件，在使用该脚本编译时可添加一个参数，例如`./build.sh 192.168.1.100`，可将编译好的程序直接发送到板卡上，要注意`PC`和板卡要连接同一网络，执行脚本时后面的`IP`地址为板卡`IP`
- 更新`build.sh`文件，在使用该脚本编译时可添加第二个参数`-r`，添加该参数后，会在编译完发送到板卡上后，直接让板卡开始运行程序，不过运行后是后台运行，如果要查看板卡运行过程中的打印信息，可以查看板卡的`/tmp/`目录下，编译完后也会有这些打印提示
- 更新`build.sh`文件，在使用该脚本编译时，如果使用的是新的虚拟机，可能会没有安装`cmake`和`pkg-confg`工具，脚本会自动检测，如果没有安装会自动安装对应缺失脚本

# 日志更新规范说明

|  模块  | 说明                                                                                                                                  |
| :--: | :---------------------------------------------------------------------------------------------------------------------------------- |
|  版本号 | 根据实时库版本更新情况描述，按时间倒序排列（最新的在最上面）                                                                                                      |
|  日期  | 统一使用`YYYY-MM-DD`格式，未发布版本标注为【未发布】                                                                                                    |
| 变更类型 | 固定分类，便于查阅：①.新增(Added)：新功能；②.更改(Changed)：现有功能调整；③.修复(Fixed)：bug修复；④.移除(Removed)：删除的功能；⑤.废弃(Deprecated)：即将移除的功能；⑥.安全(Security)：安全相关修复 |
| 描述风格 | 每条变更用`⚫`开头，使用简洁、客观的语言，聚焦【做了什么】，避免冗余                                                                                                 |
